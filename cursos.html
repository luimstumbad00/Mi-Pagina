CSS y GIT

CSS

¿Qué es?

Llamado el lenguaje de hojas en estilo de cascada, CSS llegó para complementar al HTML, debido a que HTML en crudo es muy básico y falta de dinámica, CSS tiene elementos de codificación y estilos para mejorar el diseño de páginas web.

¿Por qué surgiÓ CSS?

El CSS fue creado para trabajar en conjunto con el HTML. Y debido al concepto que utiliza (la separación de presentación y contenido), la tecnología CSS tiene un estándar muy amplio, con especificaciones, beneficios y posibilidades muy grandes. Dato curioso, esta página está hecha a base de CSS y HTML

Ventajas de usar CSS

En resumen, con CSS ahorras tiempo, al ahorrarte repetir líneas de código como en normal, hace el código más corto y menos propenso a errores, esto hace que las posibilidades de personalización sean casi infinitas. Hoy en día, esto se está volviendo una necesidad más que algo básico.

GIT

¿Qué es?

Git es un software de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia, la compatibilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente.

¿Por qué surgió CSS?

Git se ha ideado para posibilitar la ramificación y el etiquetado como procesos de primera importancia (a diferencia de SVN) y las operaciones que afectan a las ramas y las etiquetas (como la fusión o la reversión) también se almacenan en el historial de cambios.

Comandos principales:

Debido a que git, al igual que el cmd, no tiene interfaz de usuario, esta se debe manejar desde comandos; a continuación te presentamos los comandos más utilizados en este software. Todos los comandos presentados a continuación, así como todos los disponibles deben ir primero con la palabra "git".

Init

Este comando sirve para crear un nuevo repositorio, ya sea con contenido o totalmente nuevo y vacío.

Status

Este te muestra el estado y todas las modificaciones en el repositorio que se está trabajando.

Branch

Sirve para crear, enumerar o eliminar dentro de tu repositorio de trabajo.

Checkout

Sirve para desplazarte entre las ramas de tu repositorio, y al momento de extraer una rama, se actualizan todos los archivos en el directorio de trabajo.

Remote

Útil para establecer y deshacer conexiones con un repositorio remoto.

Pull

Utilizado para descargar el contenido del repositorio remoto con el que se tiene conexión y poder actualizar al instante todo el repositorio local.

Push

Una vez hecho los cambios necesarios en nuestro repositorio local, usamos este comando para subir todos nuestros cambios al repositorio remoto.

Clone

Se utiliza para clonar o copiar un repositorio remoto a uno local.

Commit

Hace una captura instantánea de todos los cambios en los repositorios.
